# Claude Code settings 說明（給 AI 參考）

此檔說明 `~/.claude/settings.json` 的重要設定與背後意圖，供未來的 AI session
快速理解使用者的環境慣例。**這是說明文件，不是可還原的備份**；實際設定以
`~/.claude/settings.json` 為準。

最後更新：2026-06-18

## 路徑約定

本文以 `<NOTE_REPO>` 代表 `Note` git repo 在當前機器上的絕對路徑（其下含 `claude/` 目錄）。
換機器或換放置位置時，只需重新認定 `<NOTE_REPO>` 指向何處，內文不必改。
查當前路徑：在 repo 內執行 `git rev-parse --show-toplevel`。

## 檔案組織慣例

使用者把 Claude Code 的設定資產放在 `<NOTE_REPO>/claude/`，再用 symlink 連回 `~/.claude/`。
已知 symlink：

- `~/.claude/skills` → `<NOTE_REPO>/claude/skills`
- `~/.claude/hooks` → `<NOTE_REPO>/claude/hooks`
- `~/.claude/CLAUDE.md` → `<NOTE_REPO>/claude/CLAUDE.md`

`~/.claude/settings.json` 本身**不是** symlink，是 `~/.claude` 下的實體檔。
`statusline/` 也在這個 repo 裡，但由 settings.json 直接以絕對路徑引用。

## 重要設定逐項說明

### autoMemoryDirectory: `<NOTE_REPO>/claude/memory`

- 內建 auto memory 的寫入位置，改指到此 repo 內，讓 auto memory 受 git 版控、
  可跨機器（透過 git）。settings.json 內以該機器的實際絕對路徑填寫。
- 預設位置原為 `~/.claude/projects/<project>/memory/`（machine-local、由 git repo 推導），
  改用此設定避開「路徑隨工作目錄漂移」與「不跨機器」的問題。

### language: `繁體中文`

- 所有輸出一律繁體中文。嚴禁簡體字。

### alwaysThinkingEnabled: `false`

- 不強制每次思考。

### hooks.PreToolUse（Bash matcher）— 重要的安全閘門

這些 hook 是「強制執行」層，不論 AI 怎麼決定都會先攔截。對應 skill 規則（cli skill）裡
的「破壞性／費用操作需確認」：

- `bq-query-gate.sh` — `bq query:*`：實際掃描資料的 BigQuery query 會計費，需確認。
- `git-publish-gate.sh` — `git:*` / `glab:*` / `gh:*`：對外發布類操作需確認。
- `kubectl-readonly-gate.sh` — `kubectl:*`：限制為唯讀。

> 對應慣例：git/glab/gh/bq query 等對外或計費操作，AI 應先說明再執行；kubectl 只做唯讀。

### statusLine

- 自訂狀態列，執行 `<NOTE_REPO>/claude/statusline/statusline-command.sh`。
